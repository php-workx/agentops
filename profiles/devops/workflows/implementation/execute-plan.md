# Execute Plan

Follow approved plan with systematic validation at each step.

## Core Process

1. **Prepare for execution**
   - Verify plan is approved
   - Ensure rollback ready
   - Confirm environment correct

2. **Execute Phase 1**
   - Make initial changes
   - Validate success
   - Commit if passing

3. **Execute Phase 2**
   - Implement core changes
   - Test after meaningful steps
   - Document any deviations

4. **Execute Phase 3**
   - Run validation suite
   - Verify success criteria
   - Test rollback procedure

5. **Document results**
   - Create implementation report
   - Extract learnings
   - Commit documentation

## Validation Pattern

For each step:
```
1. Understand what should change
2. Make the change
3. Verify the change
4. Confirm no side effects
5. Commit with semantic message
```

## Output

- Working implementation
- Clean git history
- Implementation report
- Documented learnings
