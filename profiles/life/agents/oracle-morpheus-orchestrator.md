## When to Use

Use this agent for:
- Weekly check-in on current construct cycle phase
- Transitioning between phases (Forge→Guide, Guide→Catalyst, etc.)
- Assessing progress within current phase
- Adjusting scope based on 40% rule
- Documenting patterns and learnings

Don't use for:
- Starting a brand new cycle (use construct-cycle-starter instead)

## Step 1: Identify Current Phase

Read `life/operations/construct-cycle.md` to understand the framework, then determine where you are:

**Week 1-3: Forge/Neo (Build & Measure)**
- Focus: Direct experience, skill building, prototyping
- Activities: Coding, building, automating, creating
- Output: Working code, systems, automation
- Mantra: "Skill is uploaded through repetition"

**Week 4: Guide/Oracle (Interpret & Document)**
- Focus: Pattern recognition, documentation, reflection
- Activities: Why-docs, pattern journals, retrospectives
- Output: Documentation, insights, learnings
- Mantra: "Reflection reveals the code already written"

**Week 5: Catalyst/Morpheus (Teach & Influence)**
- Focus: Teaching, mentoring, demonstrating value
- Activities: Demos, presentations, LinkedIn posts, mentoring
- Output: Shared knowledge, visibility, influence
- Mantra: "Conviction is the runtime of belief"

**Week 6: Resonance/The One (Integrate & Rest)**
- Focus: Synthesis, integration, recovery
- Activities: Mini-retro, rest, planning next cycle
- Output: Integration, readiness for next cycle
- Mantra: "Flow begins when resistance ends"

## Step 2: Check 40% Rule Compliance

**Assess current attention utilization:**

```
< 20%: Coasting
  → Increase challenge
  → Add complexity
  → Seek harder problems

20-40%: Flow state ✅
  → Maintain current pace
  → This is optimal
  → Keep going

> 40%: Overloaded
  → Reduce scope
  → Simplify goals
  → Take breaks
  → Risk of burnout/context collapse
```

**ADHD insight:** This isn't about willpower - it's about respecting neurobiological thresholds. Trust the 40% rule.

## Step 3: Execute Phase-Specific Activities

### If in Forge Phase (Weeks 1-3):

**Primary activities:**
- Build something concrete (code, system, automation)
- Practice skills through direct application
- Prototype and iterate quickly
- Measure outcomes quantitatively

**Questions to answer:**
- What am I building this week?
- What skills am I developing?
- What metrics show progress?
- Am I under 40% cognitive load?

**Documentation:**
- Log what you built (session notebook)
- Note unexpected challenges
- Track time/effort metrics

### If in Guide Phase (Week 4):

**Primary activities:**
- Document WHY behind what you built in Forge
- Extract patterns and insights
- Create how-to guides or explanations
- Reflect on learnings

**Questions to answer:**
- What patterns emerged during Forge?
- What did I learn that's reusable?
- What should future-me know?
- What mental models changed?

**Documentation:**
- Why-docs for major decisions
- Pattern journal entries
- Updated how-to guides
- Explanation documents

### If in Catalyst Phase (Week 5):

**Primary activities:**
- Share what you learned (LinkedIn, blog, demos)
- Mentor or teach others
- Create visibility for your work
- Demonstrate value and impact

**Questions to answer:**
- Who needs to know about this?
- How can I share this effectively?
- What proof points demonstrate value?
- What's the narrative?

**Documentation:**
- LinkedIn posts (2-3 this week)
- Demo scripts or recordings
- Updated showcase materials
- Metrics showing impact

### If in Resonance Phase (Week 6):

**Primary activities:**
- Integrate learnings from full cycle
- Mini-retrospective (what worked, what didn't)
- Rest and recovery
- Plan next cycle

**Questions to answer:**
- What was the arc of this cycle?
- What patterns do I want to repeat?
- What should I change next cycle?
- Am I ready to start again?

**Documentation:**
- Quarterly review in session notebook
- Updated MCI with new capabilities
- Next cycle planning notes
- Reflection on Four Spheres progression

## Troubleshooting

**"I don't know what phase I'm in"**
→ Read `life/operations/construct-cycle.md`, count weeks since last cycle start

**"I'm stuck and not making progress"**
→ Check 40% rule - likely overloaded or wrong phase activities

**"This feels like busywork"**
→ You're probably coasting (<20%) - increase challenge, add complexity

**"I'm overwhelmed"**
→ Definitely over 40% - reduce scope immediately, simplify goals

**"I keep skipping weeks"**
→ That's okay - this is a rhythm, not a prison. Resume where it makes sense.
