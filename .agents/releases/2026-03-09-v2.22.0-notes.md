# Release v2.22.0

## Highlights

This release introduces a finding-registry prevention ratchet so council
findings can be promoted into reusable planning input, a repo-native
orchestration profile for skill/runtime-aware execution, and a headless
Claude team backend for automated multi-agent flows. It also lands a
23-bug CLI audit fix-up and hardens the post-mortem closure evidence
pipeline.

## What's New

- Council findings now flow into a normalized registry with a
  prevention-ratchet loop — findings are promoted into planning and
  validation input so the same class of issue is caught earlier next time.
- Repo-profile orchestration wiring lets skills and runtimes adapt
  execution guidance based on the repository's declared profile.
- A headless Claude team backend enables automated multi-agent validation
  without interactive session scaffolding.
- 23 CLI bugs from a comprehensive audit were fixed in one pass, covering
  flag handling, output formatting, and runtime parity.
- Post-mortem closure now captures staged and worktree evidence durably,
  preventing proof loss across compaction boundaries.

## All Changes

### Added

- Finding-registry prevention ratchet with normalized council
  write/read-path support.
- Repo-native orchestration profile with skill and runtime wiring.
- Headless Claude team backend for automated multi-agent flows.

### Changed

- Codex bundles, embedded standards references, and install artifacts
  synced after merging preserved branch work.
- Validation-cycle feedback folded back into tracked learnings and
  patterns.

### Fixed

- Merged-branch fallout in `ao lookup` and inject scoring for findings.
- 23 CLI bugs from comprehensive audit (flag handling, output, runtime
  parity).
- Post-mortem closure hardening: durable proof, staged/worktree evidence,
  and canonical-root hygiene.
