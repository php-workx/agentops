## The Repo Strategy

### `12-factor-agentops/` (Public, Maintained)

**Purpose:** Methodology + Thought Leadership

**What to include:**
- Four Pillars, Five Laws
- Universal patterns (multi-phase, bundles, orchestration, routing)
- Case studies (anonymized from internal work)
- How to apply methodology

**What NOT to include:**
- Implementation details (let tools handle this)
- Plugin system (Claude Code has this)
- Installation scripts (tools handle this)

**Maintenance:** Low (methodology doesn't change often)

**Value:** Thought leadership, public visibility, career positioning

### `agentops/` (Archive or Minimal)

**Purpose:** Reference Implementation (Optional)

**Options:**
1. **Archive** - Keep as reference, don't maintain
2. **Minimal** - Plugin templates only, minimal maintenance
3. **Convert** - Transform to "How to create plugins using AgentOps" guide

**Maintenance:** Minimal or none

**Value:** Reference for methodology application
